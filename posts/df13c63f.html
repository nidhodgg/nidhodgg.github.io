包装类

介绍：根据八种数据类型相关的引用类型——包装类

  1. Integer：int的包装类
  2. Long：long的包装类
  3. Float：float的包装类
  4. Double：double的包装类
  5. Boolean：boolean的包装类
  6. Character：char的包装类
  7. Short：short的包装类
  8. Byte：byte的包装类

包装类和基本数据类型的转换

在Java中，包装类和基本数据类型之间可以进行相互转换。

  1. 自动装箱（Auto Boxing）：即将基本类型转换为对应的包装类对象。
    例如，int可以自动装箱为Integer对象。这是由Java编译器自动完成的。
  1. 自动拆箱（Auto Unboxing）：即将包装类对象转换为对应的基本类型。
    例如，Integer可以自动拆箱为int。同样，这也是由Java编译器自动完成的。
  1. 使用包装类的valueOf()方法进行转换：包装类提供了valueOf()方法，可以将基本类型转换为对应的包装类对象。
  1. 使用包装类的xxxValue()方法进行转换：包装类提供了xxxValue()方法，可以将包装类对象转换为对应的基本类型。方法名对应着基本类型
    例如intValue()、doubleValue()等。

需要注意的是，在进行包装类和基本类型的转换时，应确保数据的合法性，避免出现异常。
例如，当包装类对象为null时，进行自动拆箱或使用xxxValue()方法可能会抛出NullPointerException异常。

包装类型和 String 类型的相互转换

在Java中，包装类和String类型之间也可以进行相互转换。

  1. 使用包装类的toString()方法将包装类对象转换为String类型。
  1. 使用String类的valueOf()方法将基本类型或包装类对象转换为String类型。
  1. 使用包装类的静态valueOf()方法将String类型转换为包装类对象。
  1. 使用包装类的构造函数将String类型转换为包装类对象。
  1. 使用包装类的静态parseXxx()方法将String类型转换为基本类型。

需要注意的是，在进行String类型和包装类之间的转换时，应确保字符串内容符合对应的数据类型，否则可能会抛出NumberFormatException异常。

Integer 类和 Character 类的常用方法

Integer 类和 Character 类都是Java中的包装类，提供了许多常用的方法来操作和处理整数和字符。

Integer 类的常用方法：

  1. parseInt(String s)：将字符串解析为整数。

  2. valueOf(int i)：将整数转换为 Integer 对象。

  3. intValue()：将 Integer 对象转换为 int 类型。

  4. compareTo(Integer anotherInteger)：比较两个 Integer 对象的大小。

Character 类的常用方法：

  1. isDigit(char ch)：判断字符是否为数字。

  2. isLetter(char ch)：判断字符是否为字母。

  3. isUpperCase(char ch)：判断字符是否为大写字母。

  4. isLowerCase(char ch)：判断字符是否为小写字母。

String和StringBuffer 类

String类

用于保存字符串，也就是一串字符序列

字符串常量用双引号包裹起来（””）如”你好”，”xzl”，”重庆”

字符串的字符使用Unicode字符编码，一个字符占两个字节

注意事项

  1. String 是 final 类，不能被其他的类继承

  2. 字符串是不可变的，一个字符串一旦被分配，内容是不可变的

  3. 线程不安全，但效率高

  4. String 有属性 private final char value[]; 用于存放字符串内容

  5. value 是一个 final 类型， 不可以修改：即 value 不能指向

    新的地址，但是单个字符内容是可以变化

创建方式

  1. 直接赋值 String s = "xzljava";
    先看常量池是否有”xzljava”数据空间，如果有直接指向，如果没有重新创建然后指向，s最终指向常量池的地址空间
  2. 调用构造器 String s = new String("xzljava");
    先在堆中创建空间，里面维护了value属性，value属性指向常量池的”xzljava”空间
    如果常量池没有”xzljava”，则重新创建，如果有通过value指向
    s最终指向的是堆中的空间

常用方法

  1. length()：返回字符串的长度。

  2. charAt(int index)：返回指定位置的字符。

  3. substring(int beginIndex) 和 substring(int beginIndex, int endIndex)：返回从指定位置开始到结束的子字符串。

  4. concat(String str)：将指定字符串连接到当前字符串的末尾。

  5. equals(Object obj) 和 equalsIgnoreCase(String anotherString)：用于比较字符串是否相等（区分大小写或不区分大小写）。

  6. indexOf(int ch) 和 indexOf(String str)：返回指定字符或字符串在当前字符串中第一次出现的位置。

  7. replace(char oldChar, char newChar) 和 replace(CharSequence target, CharSequence replacement)：将指定字符或字符串替换为新的字符或字符串。

  8. toUpperCase() 和 toLowerCase()：将字符串转换为全大写或全小写。

StringBuffer 类

介绍：代表可变字符序列

特点：线程安全，可以对字符串进行修改（增删），但效率低

使用方法：与String相同，但StringBuffer 是可变长度

注意事项：StringBuffer 是一个容器

常用方法

  1. append(String str)：将指定字符串添加到StringBuffer的末尾。

  2. insert(int offset, String str)：在指定位置插入字符串。

  3. delete(int start, int end)：删除指定范围内的字符。

  4. replace(int start, int end, String str)：替换指定范围内的字符为指定字符串。

  5. reverse()：反转StringBuffer中的字符顺序。

  6. length()：返回StringBuffer的长度（字符数）。

  7. toString()：将StringBuffer对象转换为String对象。

StringBuilder 类

介绍

  1. 一个可变字符序列，可以修改字符串内容
  2. 线程不安全
  3. 效率比StringBuilder更高

常用方法与StringBuffer一样

String和StringBuffer对比

  1. 可变性：String 类是不可变的，即一旦创建了一个 String 对象，就无法修改其内容。而 StringBuffer 类是可变的，可以随时修改其内容。
  2. 线程安全性：String 类是不线程安全的，即多个线程同时访问一个 String 对象时可能会发生问题。而 StringBuffer 类是线程安全的，可以被多个线程同时访问。
  3. 性能：由于 String 类是不可变的，每次对字符串进行修改时都会创建一个新的 String 对象，这会消耗较多的内存。而 StringBuffer 类是可变的，对字符串的修改是在原对象上直接进行，不会创建新对象，因此效率更高。

如果需要频繁修改字符串，尤其是在多线程环境下，使用 StringBuffer 类。
如果字符串不需要修改或者只需少量修改，使用 String 类，它更简洁和安全。

String 和 StringBuffer 相互转换

  1. 将String转换为StringBuffer：可以使用StringBuffer的构造方法，将String作为参数传入。

  2. 将StringBuffer转换为String：可以使用StringBuffer的toString()方法，将StringBuffer对象转换为String对象。

注意事项：由于String是不可变的，而StringBuffer是可变的，当将String转换为StringBuffer时，会创建一个新的StringBuffer对象来存储原始字符串的值，并且在之后的操作中，对StringBuffer对象的修改不会影响到原始字符串的值。同样地，当将StringBuffer转换为String时，会创建一个新的String对象，该对象的值是StringBuffer所包含的字符序列。

进行String和StringBuffer之间的转换时，需要考虑到对象的可变性和性能的影响。

String、StringBuffer 和 StringBuilder 的比较

  1. 可变性：String是不可变的，一旦创建就不能被修改。而StringBuffer和StringBuilder都是可变的，可以通过方法来修改字符串的内容。

  2. 线程安全性：String是线程安全的，因为它是不可变的，多个线程可以共享同一个String对象而不需要担心修改冲突。而StringBuffer是线程安全的，它的方法都是同步的，可以安全地在多线程环境中使用。StringBuilder是非线程安全的，它的方法没有同步机制，不适用于多线程环境。

  3. 性能：由于String是不可变的，每次对String进行修改都会创建一个新的String对象，造成了额外的内存开销。而StringBuffer和StringBuilder是可变的，可以直接修改原字符串，不会产生额外的内存开销。在大量字符串拼接或修改的情况下，StringBuilder的性能通常比StringBuffer更好，因为StringBuilder没有同步开销。

多线程环境下频繁地拼接或修改字符串，使用StringBuffer。
单线程环境下频繁地拼接或修改字符串，使用StringBuilder
字符串不需要修改或者只进行少量的修改操作，使用String

总结：

String：不可变字符序列，效率低

StringBuffer：可变字符序列，效率高，线程安全

StringBuilder：可变字符序列，效率最高，线程不安全

BigInteger 和 BigDecimal 类

介绍

BigInteger和BigDecimal是Java中的两个用于处理大数和高精度数的类。

BigInteger类：用于表示任意大小的整数，它提供了大整数的运算和操作。

BigDecimal类：用于表示任意大小和精度的十进制数，它提供了高精度的运算和操作。

常用方法

BigInteger常用方法

  1. add(BigInteger val)：将当前BigInteger对象与val相加。

  2. subtract(BigInteger val)：将当前BigInteger对象减去val。

  3. multiply(BigInteger val)：将当前BigInteger对象与val相乘。

  4. divide(BigInteger val)：将当前BigInteger对象除以val，返回商。

  5. mod(BigInteger val)：将当前BigInteger对象除以val，返回余数。

BigDecimal常用方法

  1. add(BigDecimal val)：将当前BigDecimal对象与val相加。

  2. subtract(BigDecimal val)：将当前BigDecimal对象减去val。

  3. multiply(BigDecimal val)：将当前BigDecimal对象与val相乘。

  4. divide(BigDecimal val)：将当前BigDecimal对象除以val，返回商。

  5. remainder(BigDecimal val)：将当前BigDecimal对象除以val，返回余数。

日期类

介绍

Java中的日期类主要有两个：Date和Calendar。在Java 8及以后的版本中，还引入了新的日期和时间API，包括LocalDate、LocalTime、LocalDateTime等。

Date类介绍和常用方法

Date类用于表示特定的时间点，以毫秒级别的精度表示时间。

使用流程

常用方法

  • Date()：无参构造方法，创建一个表示当前时间的Date对象。

  • getTime()：返回以毫秒为单位表示的日期。

  • toString()：将Date对象转换为字符串表示。

  • compareTo(Date anotherDate)：比较当前Date对象与另一个Date对象的顺序。

Calendar类介绍和常用方法

Calendar类是一个抽象类，提供了对日期和时间进行各种操作的方法。通过Calendar类，可以进行日期的计算、格式化等操作。

使用流程

常用方法

  • getInstance()：获取默认时区的Calendar对象。

  • get(int field)：获取指定字段的值，如年、月、日等。

  • set(int field, int value)：设置指定字段的值。

  • add(int field, int amount)：在指定字段上增加或减少指定的值。

LocalDate类介绍和常用方法

Java 8引入的新日期API中的LocalDate类表示不带时区的日期。

使用流程

常用方法

  • now()：获取当前日期。

  • getYear()、getMonthValue()、getDayOfMonth()：获取年、月、日。

  • plusYears()、plusMonths()、plusDays()：在指定字段上增加指定的值。

  • compareTo(LocalDate otherDate)：比较当前日期与另一个日期的顺序。
