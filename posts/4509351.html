集合和数组的区别，及其介绍

数组

  • 长度必须指定，一旦指定无法更改
  • 保存的必须是同一类型的元素
  • 使用数组增加和删除元素的代码繁琐

集合

  • 可以动态保存多个对象-方便
  • 提供了一系列方便操作对象的方法
  • 使用集合添加和删除元素的代码简单清晰

创建集合

ArrayList：ArrayList arrayList = new ArrayList();
添加：arrayList.add("jack");arrayList.add("tom");

HashMap：HashMap hashMap = new HashMap();
添加：hashMap.put("NO1", "北京");hashMap.put("NO2", "上海");

集合的框架

集合主要分为两大类：Collection和Map

Collection接口有两个重要的子接口：List和Set
List包括ArrayList，Vector，LinkedList
Set包括HashSet，LinkedHashSet
他们实现的子类都是单列集合

Map接口的实现子类都是双列集合，存放的是K-V键值对
Map包括HashMap，Hashtable

Collection接口

特点

  • Collection接口实现子类可以存放多个元素，每个元素都可以是Object
  • Collection的实现类，有些可以存放重复的元素，有些不行
  • Collection的实现类，有些是有序的，有些是无序的
  • Collection接口没有直接实现的子类，都是通过List和Set来实现的

List：有序，可以存放重复的元素

  • ArrayList
  • Vector
  • LinkedList

Set：无序，不可以存放重复的元素

  • TreeSet
  • HashSet

方法

  • 增：add
  • 删：remove
  • 改：set
  • 查：get
  • 翻转顺序：reverse(List)
  • 随机排序：shuffle(List)
  • 升序排序：sort(List)
  • 根据指定的Comparator排序：sort(List，Comparator)
  • i，j元素交换：swap(List，int，int)

遍历方式

  1. 使用 Iterator(迭代器)

    1. 得到对应的迭代器
      Iterator iterator = xxx.iterator();

    2. 使用while循环遍历（快捷键itit）

    3. . 如果希望再次遍历，需要重置迭代器

  2. 增强 for 循环

（快捷键 I ）

注意：增强for还可以用于数组

增强for底层就是迭代器
增强for就是简化版本的迭代器

  1. 普通for循环

(快捷键fori)

List接口

特点：

  • 包含ArrayList，Vector，LinkedList等类型

  • List集合类中的元素有序（添加和取出顺序一致），且可重复
    List list = new ArrayList();
    list.add("jack");
    list.add("xzl");

  • 每个元素都有对应的顺序索引，即支持索引
    System.out.println(list.get(1));//xzl
  • 元素都对应一个整数型序号在容器中的位置，可以根据需要存取容器中的元素

使用流程：

  1. 创建List集合
    List list = new ArrayList();
    List list = new Vector();
    List list = new LinkedList();
  2. 增删改查（add，remove，set，get）
  3. 遍历（增强for循环，Iterator迭代器，普通for循环）

常用方法：

  • add(int index, Object ele)：在 index 位置插入 eles 元素
    没带索引自动加到最后

  • boolean addAll(int index, Collection eles)：从 index 位置开始将 eles 中的所有元素添加进来

  • Object get(int index)获取指定 index 位置的元素

  • int indexOf(Object obj)返回 obj 在集合中首次出现的位置

  • int lastIndexOf(Object obj)返回 obj 在当前集合中末次出现的位置

  • Object remove(int index)移除指定 index 位置的元素，并返回此元素

  • Object set(int index, Object ele)设置指定 index 位置的元素为 ele , 相当于是替换.

  • List subList(int fromIndex, int toIndex)返回从 fromIndex 到 toIndex 位置的子集合

ArrayList

特点：

  • ArrayList可以加入null,并且可以加入多个
  • ArrayList是由数组来实现数据存储的
  • ArrayList基本等同于Vector
    ArrayList线程不安全，多线程不建议使用
    Vector线程安全，多线程使用

ArrayList源码分析：

  • ArrayList中维护了一个Object类型的数组elementData

    transient Object[] elementData;//transient表示瞬间，短暂的，表示该属性不会被序列号

  • 当创建ArrayList对象时，如果使用的是无参构造器，则初始elementData容量为0，第1次添加，则扩容elementData为10，如需要再次扩容，则扩容elementData为1.5倍。

  • 如果使用的是指定大小的构造器，则初始elementData容量为指定大小，如果需要扩容，则直接扩容elementData为1.5倍。

Vector

特点：

  • 底层也是一个对象数组
  • 线程同步的，也就是线程安全的
  • 操作方法带有synchronized关键字

Vector源码分析：

  • Vector中维护了一个Object类型的数组elementData

    transient Object[] elementData;//transient表示瞬间，短暂的，表示该属性不会被序列号

  • 当创建Vector对象时，如果使用的是无参构造器，则初始elementData容量为10，扩容elementData为2倍。

  • 如果使用的是指定大小的构造器，则初始elementData容量为指定大小，如果需要扩容，则直接扩容elementData为2倍。

LinkedList

特点：

  • 底层实现了双向链表，双端队列
  • 可以添加任意元素（可以重复），包括null
  • 线程不安全，线程不同步

源码分析：

  • 底层维护一个双向链表
  • 两个属性first和last分别指向首节点和尾节点
  • 每个节点（Node对象）里面又维护了prev，next，item三个属性
    通过prev指向前一个，next指向后一个，最终实现双向链表
  • 元素的添加和删除，不是通过数组完成的，效率高

Vector和ArrayList对比

底层结构 版本 线程安全/效率 扩容倍数
ArraList 可变数组 jdk1.2 不安全，效率高 无参，默认0，第一次10，第二次1.5倍扩容；指定大小1.5倍
Vector 可变数组 jdk1.0 安全，效率低 无参，默认10，满后，2倍扩容；指定大小，每次都2倍扩容

多线程，选Vector

LinkedList和ArrayList对比

底层结构 增删效率 改查效率
ArrayList 可变数组 效率低，数组扩容 效率高
LinkedList 双向链表 效率高，链表追加 效率低

改查多，选ArrayList
增删多，选LinkedList
大部分时间都在查询所以选择ArrayList更多

Set接口

特点：

  • 无序（添加和取出的顺序不一致），没有索引
  • 不允许添加重复元素，最多包含一个null
  • 包括HashSet和TreeSet等

方法：和 List 接口一样, Set 接口也是 Collection 的子接口，因此，常用方法和 Collection 接口一样

  • 增：add
  • 删：remove
  • 改：set
  • 查：get

遍历方式：

  1. 迭代器
  2. 增强for循环

HashSet

特点：

  • 实现Set接口
  • 底层实际上是HashMap
  • 可以存放null，但是只能一个
  • 不保证元素是有序的
  • 不能有重复元素/对象

源码分析：

  1. 底层是HashMap
  2. 添加一个元素时，先得到hash值，后转成索引值
  3. 找到存贮数据表table，看索引位置是否已经存放有元素
  4. 如果没有，直接加入
  5. 如果有，调用equals进行比较，如果相同，放弃添加；如果不同，添加到最后
  6. 当一条链表的元素个数到达8时且table大小>=64时，树化（红黑树）

扩容机制：

  1. 第一次添加时，table扩容到16，临界值（threshold）是16*0.75=12
  2. 如果table到了临界值12，就开始扩容到16*2=32，新的临界值就是320*0.75=24
  3. 当链表元素到达8以上且table大小>=64就开始树化

LinkedHashSet

特点：

  • 是HashSet的子类
  • 底层是一个LinkedHashMap
  • 底层维护了一个数组+双向链表
  • 根据hashcode值来决定元素存放位置，用链表维护元素次序
  • 不允许添加重复元素

源码分析：

  1. 维护一个hash表和双向链表
  2. 每个节点有before和after属性，以形成双向链表
  3. 添加元素时，先求hash值，再求索引，确定元素在table的位置
    然后添加元素到双向链表，如果有元素存在，不添加；如果不存在，加到最后
  4. 遍历时，插入顺序和遍历顺序一致

Map接口

特点：

  • Map与Collection并列存在。用于保存具有映射关系的数据：Key-Value
  • Map中的key和value可以是任何引用类型的数据，会封装到HashMap$Node对象中
  • Map中的key不允许重复，原因和HashSet一样
  • Map中的value可以重复
  • Map的key可以为null，value也可以为null，注意key为null，只能有一个，value为null，可以多个
  • 常用String类作为Map的key
  • key和value之间存在单向一对一关系，即通过指定的key总能找到对应的value

比如：Map map = new HashMap();
map.put("xzl","java")

常用方法：

  • put：增 map.put("xzl","java")
  • remove：删：根据键删除映射关系
  • get：查：根据键获取值
  • size：获取元素个数
  • isEmpty:判断个数是否为 0
  • clear：清除 k-v

遍历方式：

  • containsKey：查找键是否存在
  • keyset：获取所有的键
  • entrySet：获取所有关系k-v
  • values：获取所有的值

可以使用所有的 Collections 遍历方法

  • keyset获取所有的键，再通过键获取值

  • EntrySet获取k-v

  • values取出

HashMap

特点

  1. HashMap是基于哈希表实现的Map接口的一个类，其底层采用数组加链表的存储结构。
  2. HashMap的核心成员变量有两个：table和size，table是一个Entry数组，用于存储键值对；size是HashMap中键值对的数量。
  3. 线程不安全
  4. key不能重复，值可以重复，可以使用null值和键
  5. 添加相同的key，会覆盖原来的key-val，等于修改（key不会变，val会变）
  6. 当添加key-val时，通过key的哈希值得到在table的索引。然后判断该索引处是否有元素，如果没有元素直接添加。如果该索引处有元素，继续判断该元素的key和准备加入的key相是否等，如果相等，则直接替换val;如果不相等需要判断是树结构还是链表结构，做出相应处理。如果添加时发现容量不够，则需要扩容。
  7. 第1次添加，则需要扩容table容量为16,临界值(threshold)为12(16*0.75)
  8. 以后再扩容，则需要扩容table容量为原来的2倍(32)，临界值为原来的2倍，即24依次类推
  9. 在Java8中，如果一条链表的元素个数超过TREEIFY_THRESHOLD（默认是8）,并且table的大小>= MIN_TREEIFY_CAPACITY(默认64),就会进行树化(红黑树)

常用方法

  • put：添加键值对
  • get：获取键对应的值
  • containsKey：判断HashMap是否包含指定键
  • containsValue：判断HashMap是否包含指定值
  • keySet：获取HashMap的键集合
  • values：获取HashMap的值集合
  • remove：删除指定键的键值对
  • isEmpty：判断HashMap是否为空
  • size：获取HashMap的大小
  • clear：清空HashMap

源码分析

  1. HashMap的初始化，常用的是无参构造函数，它会初始化一个默认大小的数组。

  2. put方法：用于插入键值对。
    当插入一个键值对时，首先根据键的hashCode计算出在数组table中的位置，然后遍历该位置上的链表，找到对应的键值对
    如果找到了，则更新该键值对的值；如果没有找到，则创建一个新的节点，并将其插入到链表的头部。
    如果链表的长度达到了一定阈值（TREEIFY_THRESHOLD），则会将链表转换为红黑树。

  3. get方法：用于根据键k获取值v。
    根据键的hashCode计算出在数组中的位置，然后遍历该位置上的链表，找到对应的键值对，并返回其值。

  4. removeNode方法：删除键值对的方法
    如果要删除的节点是链表的头节点，将该节点从链表中移除，并将该索引位置的头节点指向链表的下一个节点。

    如果要删除的节点不是链表的头节点，将该节点从链表中移除，即将该节点的前一个节点的next指针指向该节点的下一个节点。

  5. 扩容机制：resize方法

Hashtable

特点

  1. Hashtable的键和值都不能为null，否则抛异常NullPointerException（空指针异常）
  2. 使用方法与HashMap一致
  3. 线程安全

HashMap和Hashtable对比

版本 线程安全（同步） 效率 运行null键null值
HashMap 1.2 不安全 高 允许
Hashtable 1.0 安全 低 不允许

多线程开发使用Hashtable

Properties

特点

1.Properties类继承自Hashtable类并且实现了Map接口，也是使用一种键值对的形式来保存数据。

2.他的使用特点和Hashtable类似

3.Properties还可以用于从xxx.properties文件中，加载数据到Properties类对象，并进行读取和修改

4.xxx.properties文件通常作为配置文件

集合使用场景分析

  1. 判断存储类型（一组对象[单列]或一组键值对[双列]）

  2. 一组对象[单列]：Collection接口

    允许重复：List

    增删多：LinkedList 双向链表

    改查多：ArrayList 可变数组

    不允许重复：Set

    无序：HashSet 数组+链表+红黑树

    排序：TreeSet

    插入和取出顺序一致：LinkedHashSet 数组+双向链表

  3. 一组键值对[双列]：Map接口

    键无序：HashMap 数组+链表

    键排序：TreeMap

    键插入和取出顺序一致：LinkedHashMap

    读取文件：Properties
