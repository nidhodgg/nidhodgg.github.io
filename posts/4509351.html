集合和数组的区别，及其介绍

数组

  • 长度必须指定，一旦指定无法更改
  • 保存的必须是同一类型的元素
  • 使用数组增加和删除元素的代码繁琐

集合

  • 可以动态保存多个对象-方便
  • 提供了一系列方便操作对象的方法
  • 使用集合添加和删除元素的代码简单清晰

创建集合

ArrayList：ArrayList arrayList = new ArrayList();
添加：arrayList.add("jack");arrayList.add("tom");

HashMap：HashMap hashMap = new HashMap();
添加：hashMap.put("NO1", "北京");hashMap.put("NO2", "上海");

集合的框架

集合主要分为两大类：Collection和Map

Collection接口有两个重要的子接口：List和Set
List包括ArrayList，Vector，LinkedList
Set包括HashSet，LinkedHashSet
他们实现的子类都是单列集合

Map接口的实现子类都是双列集合，存放的是K-V键值对
Map包括HashMap，Hashtable

Collection接口

特点：

  • Collection接口实现子类可以存放多个元素，每个元素都可以是Object
  • Collection的实现类，有些可以存放重复的元素，有些不行
  • Collection的实现类，有些是有序的，有些是无序的
  • Collection接口没有直接实现的子类，都是通过List和Set来实现的

List：有序，可以存放重复的元素

  • ArrayList
  • Vector
  • LinkedList

Set：无序，不可以存放重复的元素

  • TreeSet
  • HashSet

方法：

  • 增：add
  • 删：remove
  • 改：set
  • 查：get

遍历方式：

  1. 使用 Iterator(迭代器)

    1. 得到对应的迭代器
      Iterator iterator = xxx.iterator();

    2. 使用while循环遍历（快捷键itit）

    3. . 如果希望再次遍历，需要重置迭代器

  2. 增强 for 循环

（快捷键 I ）

注意：增强for还可以用于数组

增强for底层就是迭代器
增强for就是简化版本的迭代器

  1. 普通for循环

(快捷键fori)

List接口

特点：

  • 包含ArrayList，Vector，LinkedList等类型

  • List集合类中的元素有序（添加和取出顺序一致），且可重复
    List list = new ArrayList();
    list.add("jack");
    list.add("xzl");

  • 每个元素都有对应的顺序索引，即支持索引
    System.out.println(list.get(1));//xzl
  • 元素都对应一个整数型序号在容器中的位置，可以根据需要存取容器中的元素

使用流程：

  1. 创建List集合
    List list = new ArrayList();
    List list = new Vector();
    List list = new LinkedList();
  2. 增删改查（add，remove，set，get）
  3. 遍历（增强for循环，Iterator迭代器，普通for循环）

常用方法：

  • add(int index, Object ele)：在 index 位置插入 eles 元素
    没带索引自动加到最后

  • boolean addAll(int index, Collection eles)：从 index 位置开始将 eles 中的所有元素添加进来

  • Object get(int index)获取指定 index 位置的元素

  • int indexOf(Object obj)返回 obj 在集合中首次出现的位置

  • int lastIndexOf(Object obj)返回 obj 在当前集合中末次出现的位置

  • Object remove(int index)移除指定 index 位置的元素，并返回此元素

  • Object set(int index, Object ele)设置指定 index 位置的元素为 ele , 相当于是替换.

  • List subList(int fromIndex, int toIndex)返回从 fromIndex 到 toIndex 位置的子集合

ArrayList

特点：

  • ArrayList可以加入null,并且可以加入多个
  • ArrayList是由数组来实现数据存储的
  • ArrayList基本等同于Vector
    ArrayList线程不安全，多线程不建议使用
    Vector线程安全，多线程使用

ArrayList源码分析：

  • ArrayList中维护了一个Object类型的数组elementData

    transient Object[] elementData;//transient表示瞬间，短暂的，表示该属性不会被序列号

  • 当创建ArrayList对象时，如果使用的是无参构造器，则初始elementData容量为0，第1次添加，则扩容elementData为10，如需要再次扩容，则扩容elementData为1.5倍。

  • 如果使用的是指定大小的构造器，则初始elementData容量为指定大小，如果需要扩容，则直接扩容elementData为1.5倍。

Vector

特点：

  • 底层也是一个对象数组
  • 线程同步的，也就是线程安全的
  • 操作方法带有synchronized关键字

Vector源码分析：

  • Vector中维护了一个Object类型的数组elementData

    transient Object[] elementData;//transient表示瞬间，短暂的，表示该属性不会被序列号

  • 当创建Vector对象时，如果使用的是无参构造器，则初始elementData容量为10，扩容elementData为2倍。

  • 如果使用的是指定大小的构造器，则初始elementData容量为指定大小，如果需要扩容，则直接扩容elementData为2倍。

LinkedList

特点：

  • 底层实现了双向链表，双端队列
  • 可以添加任意元素（可以重复），包括null
  • 线程不安全，线程不同步

源码分析：

  • 底层维护一个双向链表
  • 两个属性first和last分别指向首节点和尾节点
  • 每个节点（Node对象）里面又维护了prev，next，item三个属性
    通过prev指向前一个，next指向后一个，最终实现双向链表
  • 元素的添加和删除，不是通过数组完成的，效率高

Vector和ArrayList对比

底层结构 版本 线程安全/效率 扩容倍数
ArraList 可变数组 jdk1.2 不安全，效率高 无参，默认0，第一次10，第二次1.5倍扩容；指定大小1.5倍
Vector 可变数组 jdk1.0 安全，效率低 无参，默认10，满后，2倍扩容；指定大小，每次都2倍扩容

多线程，选Vector

LinkedList和ArrayList对比

底层结构 增删效率 改查效率
ArrayList 可变数组 效率低，数组扩容 效率高
LinkedList 双向链表 效率高，链表追加 效率低

改查多，选ArrayList
增删多，选LinkedList
大部分时间都在查询所以选择ArrayList更多

Set接口

特点：

  • 无序（添加和取出的顺序不一致），没有索引
  • 不允许添加重复元素，最多包含一个null
  • 包括HashSet和TreeSet等

方法：和 List 接口一样, Set 接口也是 Collection 的子接口，因此，常用方法和 Collection 接口一样

  • 增：add
  • 删：remove
  • 改：set
  • 查：get

遍历方式：

  1. 迭代器
  2. 增强for循环

HashSet

特点：

  • 实现Set接口
  • 底层实际上是HashMap
  • 可以存放null，但是只能一个
  • 不保证元素是有序的
  • 不能有重复元素/对象

源码分析：

  1. 底层是HashMap
  2. 添加一个元素时，先得到hash值，后转成索引值
  3. 找到存贮数据表table，看索引位置是否已经存放有元素
  4. 如果没有，直接加入
  5. 如果有，调用equals进行比较，如果相同，放弃添加；如果不同，添加到最后
  6. 当一条链表的元素个数到达8时且table大小>=64时，树化（红黑树）

扩容机制：

  1. 第一次添加时，table扩容到16，临界值（threshold）是16*0.75=12
  2. 如果table到了临界值12，就开始扩容到16*2=32，新的临界值就是320*0.75=24
  3. 当链表元素到达8以上且table大小>=64就开始树化

LinkedHashSet

特点：

  • 是HashSet的子类
  • 底层是一个LinkedHashMap
  • 底层维护了一个数组+双向链表
  • 根据hashcode值来决定元素存放位置，用链表维护元素次序
  • 不允许添加重复元素

源码分析：

  1. 维护一个hash表和双向链表
  2. 每个节点有before和after属性，以形成双向链表
  3. 添加元素时，先求hash值，再求索引，确定元素在table的位置
    然后添加元素到双向链表，如果有元素存在，不添加；如果不存在，加到最后
  4. 遍历时，插入顺序和遍历顺序一致

Map接口

特点：

  • Map与Collection并列存在。用于保存具有映射关系的数据：Key-Value
  • Map中的key和value可以是任何引用类型的数据，会封装到HashMap$Node对象中
  • Map中的key不允许重复，原因和HashSet一样
  • Map中的value可以重复
  • Map的key可以为null，value也可以为null，注意key为null，只能有一个，value为null，可以多个
  • 常用String类作为Map的key
  • key和value之间存在单向一对一关系，即通过指定的key总能找到对应的value

比如：Map map = new HashMap();
map.put("xzl","java")

常用方法：

  • put：增 map.put("xzl","java")
  • remove：删：根据键删除映射关系
  • get：查：根据键获取值
  • size：获取元素个数
  • isEmpty:判断个数是否为 0
  • clear：清除 k-v

遍历方式：

  • containsKey：查找键是否存在
  • keyset：获取所有的键
  • entrySet：获取所有关系k-v
  • values：获取所有的值

可以使用所有的 Collections 遍历方法

  • keyset获取所有的键，再通过键获取值

  • EntrySet获取k-v

  • values取出

HashMap

源码分析：

  1. HashMap是基于哈希表实现的Map接口的一个类，其底层采用数组加链表的存储结构。
  2. HashMap的核心成员变量有两个：table和size。table是一个Entry数组，用于存储键值对；size是HashMap中键值对的数量。
  3. HashMap的put方法用于向HashMap中插入键值对。
  • 首先计算key的哈希值，通过hashCode方法。
  • 然后根据哈希值计算key在数组中的索引位置，通过hash方法。
  • 如果该索引位置上没有元素，则直接插入键值对。
  • 如果该索引位置上已经存在元素，即出现了哈希冲突，则使用链表的方式解决冲突。
  • 如果链表中已经存在了相同的key，则更新value值。
  • 如果链表中不存在相同的key，则将新的键值对插入到链表的末尾。
  • 如果链表的长度超过了阈值（默认为8），则将链表转换为红黑树以提高查询效率。
  • 最后检查是否需要扩容，如果超过了负载因子（默认为0.75），则进行扩容操作。
